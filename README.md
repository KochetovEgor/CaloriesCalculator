# Калькулятор калорий

Сервис позволяет пользователю указывать свой дневной рацион и считать потребление калорий, белков, жиров и углеводов (КБЖУ).

Используемые технологии:
- PostgreSQL (хранилище данных)
- Docker (контейнеризация)
- pgx (драйвер для работы с PostgreSQL)

Сервис написан с применением чистой архитектуры, в нём выделены основные слои: domain, service, storage, controller. В сервисе реализовано логирование с помощью пакета "log/slog" и аутентификация пользователей с помощью JWT.

## Запуск

Запустить сервис можно с помощью команды `docker compose up`.

## Endpoints

Backend приложения реализован в виде HTTP API. Далее описаны некоторые endpoint'ы HTTP контроллера.

**В будущем планируется переработать endpoint'ы и добавить OpenAPI документацию для удобства пользователей.**

### POST /login

Позволяет авторизоваться в приложении. Возвращает JWT, который необходимо прикладывать почти ко всем другим эндпоинтам.

### POST /register

Позволяет пользователю зарегистрироваться в приложении.

### POST /product/add

Позволяет пользователю добавить какую-либу еду из его рациона, указав её КБЖУ.

### DELETE /product/delete

Позволяет удалить еду по её названию.

### GET /product

Возвращает массив, содержащий всю добавленную пользователем еду.

### POST /ration/add

Позволяет добавить данные о дневном рационе, указав съеденную за день еду.

### POST /ration/delete

Удаляет данные о рационе за определённый день.

### GET /ration

Возвращает массив с данным о рационе за все заполненные пользователем дни.
